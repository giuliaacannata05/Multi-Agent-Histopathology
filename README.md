# Multi-Agent Histopathology System
An End-to-End Computational Pathology Prototype for Morphological Extraction and Clinical Reporting.

## Project Description
This repo holds the code for a multi-agent computational pathology prototype I built for my final thesis and internship.

The system overcomes the traditional reasoning limitations of smaller-scale multimodal models by applying the engineering principle of **Separation of Concerns**. The pipeline is split into two main players:

1. **Visual Agent (The Eyes - BLIP-2):** A fine-tuned model specialized in the histopathology domain, tasked with analyzing slide pixels (H&E staining) and extracting a purely objective morphological analysis of cellular and stromal populations.
2. **Clinical Agent (The Brain - Qwen2.5-Coder-32B-Instruct):** A Large Language Model with high parametric capacity operating as a cognitive orchestrator. It receives the textual extraction from the visual module and applies its clinical deductive reasoning (*Zero-Shot Reasoning Transfer*) to answer specific user queries and draft a structured pathology report.

The entire orchestration and execution cycle (*Thought-Code-Observation*) are governed by the **smolagents** framework by Hugging Face.

*Note: The `pipeline_config.json` and `prompts.yaml` files are auto-generated upon running the notebook to maximize code portability.*

## How to Run the Prototype
1. Open the `PipelineFinale_GiuliaCannata_ISTO.ipynb` file within a Jupyter Notebook or Google Colab environment.
2. Ensure you have enabled a runtime with GPU hardware acceleration (e.g., NVIDIA Tesla T4).
3. Execute the cells sequentially from top to bottom.
4. Click on the public link generated by **Gradio** to open the web application in your browser.
